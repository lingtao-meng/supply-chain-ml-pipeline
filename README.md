# 供应链延迟交付风险预测

用 DataCo 全球供应链数据（180K+ 条订单，53 个原始字段）做的一个延迟交付预测项目。从数据清洗、特征工程到多模型对比、SHAP 解释、业务量化，走了一遍完整的 ML 流水线。

最后 AUC 定在 0.74。不是模型的问题——下面会解释为什么。

## 业务问题

订单延迟交付是供应链里代价很高的风险。想法很简单：在订单发货之前，预测它有多大可能会延迟。如果能提前知道高风险订单，就可以加急配送、主动联系客户。

## 数据

DataCo Supply Chain Dataset，Kaggle 上公开的。180,519 条订单，53 个原始字段，包括订单类型、运输方式、产品品类、客户地区、订单日期等等。有缺失值也有异常值，比较接近真实场景。

## 做了什么

1. **数据清洗**：删了邮箱、密码、人名这种无关列。比较关键的一步是识别并移除了数据泄露特征——Delivery Status 跟标签直接相关，Days for shipping (real) 是事后才知道的，这些留着的话 AUC 能到 1.0 但那不叫预测
2. **特征工程**：从日期提取了月份、星期、季度、是否周末；算了订单总价、折扣金额；做了交互特征（客户类型×运输方式、市场×运输方式等）
3. **模型对比**：用时间切分（早期数据训练，后期数据测试），对比了 6 个模型
4. **对抗验证**：训练一个分类器来区分早期订单和后期订单，发现 AUC 高达 0.91——说明训练集和测试集的数据分布差异很大
5. **SHAP 解释**：看了哪些特征对预测贡献最大
6. **AutoGluon 天花板测试**：用 AutoML 自动跑了 10 分钟，看这个数据集理论上限在哪

## 6 个模型的结果

| 模型 | AUC | F1 |
|------|:--:|:--:|
| XGBoost | 0.744 | 0.690 |
| LightGBM | 0.741 | 0.662 |
| Random Forest | 0.742 | 0.662 |
| MLP（3层） | 0.736 | 0.654 |
| TabNet（Google DL） | 0.728 | 0.655 |
| Logistic Regression | 0.723 | 0.638 |

XGBoost 最好，AUC 0.744。TabNet 反而没打过 XGBoost——表格数据上深度学习打不过梯度提升这件事，NeurIPS 2022 的 benchmark 也有类似结论。

## 为什么 AUC 只有 0.74

这个 AUC 看起来不高，但背后有一条完整的诊断链条：

**第一层：是不是数据清洗出问题了？**

不是。我试了不同程度的清洗——从只删 PII 到删光所有可能泄露的特征——非泄露特征的 AUC 全部落在 0.72-0.73 之间，差距不到 0.01。清洗过程是干净的。

**第二层：是不是特征工程不够？**

不是。我加了时间特征、业务特征、交互特征（客户×运输、市场×运输、折扣×金额等），特征从 17 列扩到 44 列。所有特征工程的 AUC 提升加起来不到 0.01。不是特征不够，是原始特征的预测力本身就很弱——最强单特征的互信息只有 0.13。

**第三层：是不是时间漂移？**

是的。对抗验证 AUC=0.91 证明训练集和测试集的分布严重不同。我试了 6 种方法对抗时间漂移：样本时间加权、添加绝对时间特征、丢弃旧数据只用近期数据、去掉季节性特征、连续特征分箱——全部无效，AUC 提升全部小于 0.01。

**第四层：能不能用知识蒸馏？**

不能。训练了一个能看到泄露特征的教师模型（AUC 1.0），但它的知识没法传递给学生模型。因为教师学到的信号来自「实际运输天数」这种事后才知道的信息——这些信息在学生的特征里根本不存在。蒸馏传不了不存在的东西。

**为什么 Standard Class 是元凶？**

按运输方式拆开看，First Class 的 AUC 能到 0.89，Same Day 也能到 0.69。但 Standard Class 占了 60% 的数据，AUC 只有 0.54——基本上就是随机猜。这个类别里运输天数的波动太大，用当前的可用特征根本没法区分哪单会延迟、哪单不会。

**结论：0.74 是这个数据集 + 时间切分约束下的真实天花板。**

AutoGluon（AutoML）跑了 10 分钟试了多种模型和集成，最好也只做到 0.735——比我的手调 XGBoost 还低一点。连机器自己都突破不了，说明不是人的问题。

## 对抗验证的细节

训练一个 XGBoost 分类器，标签是「这条订单来自训练期还是测试期」，特征和主模型完全一样。结果这个分类器的 AUC 达到 0.91。

为什么这么高？因为产品品类（Category）和市场区域（Market）在不同时间段的分布变了——训练期某几类产品多，测试期另几类产品多。这意味着训练集里学到的东西，在测试集上不一定适用。

大多数课程项目不会做这个检查——他们用随机切分，训练和测试的数据分布一样，AUC 自然高。但真正的生产环境里，你永远是用过去的数据预测未来。

## 运行

数据文件比较大（93MB），需要先从 Kaggle 下载：

```bash
python -c "import kagglehub; kagglehub.dataset_download('evilspirit05/datasupplychain')"
```

然后把 CSV 放到 data/ 目录下。

```bash
pip install pandas numpy scikit-learn xgboost lightgbm shap imbalanced-learn streamlit
cd notebooks && jupyter notebook supply_chain_ml_pipeline.ipynb
```
